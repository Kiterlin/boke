暗色模式经常被低估。它不是附加主题，而是一套完整的视觉质量检查：颜色 token 是否可靠、边界是否清楚、阴影是否仍然成立、代码块是否可读。

## 对比度优先于氛围

暗色界面很容易变成一片低对比灰。正文、次级文本、边框和背景必须有明确层级，否则阅读会很累。

## 阴影需要改成边界和光感

浅色模式里的投影在暗色模式里通常不成立。更稳妥的方式是使用边框、细微高光和背景层级来表达卡片关系。

## 代码块必须单独校准

代码块不是普通正文。它需要固定宽度字体、足够行高、清楚的背景和横向滚动能力。

```tsx
export function ThemeToggle() {
  return <button aria-label="切换暗色模式">Toggle</button>;
}
```
